Interface and Abstraction
Task 1: Design a media player system using both an interface and an abstract class. Begin by creating an interface named Playable that declares two methods: play() and pause(). Then, implement an abstract class called MediaFile that includes a String title field, a constructor to initialize the title, and a concrete implementation of the pause() method that prints a message indicating the media is paused. This abstract class should also declare an abstract method showDetails() to be implemented by subclasses. Create two subclasses: AudioFile and VideoFile, each providing their own implementation of play() and showDetails() to reflect their specific media type. In the main method, instantiate objects of both classes and demonstrate their functionality by calling all relevant methods.

Task 2:Create a course management system that combines interface and abstraction. Start by defining an interface named Enrollable with two methods: enroll() and complete(). Then, implement an abstract class called Course that includes a String courseName field, a constructor to initialize it, and a concrete complete() method that prints a message indicating the course has been completed. The abstract class should also declare an abstract method showContent() to be implemented by subclasses. Develop two subclasses: ProgrammingCourse and DesignCourse, each implementing the enroll() and showContent() methods with course-specific behavior. In the main method, create instances of both course types, enroll in them, and display their content and completion messages.

Task 3: Create an authentication system using abstraction where Admin and Guest classes override the abstract authenticate method to provide different login validation behaviours .The main method checks user input against both types to determine login success.
